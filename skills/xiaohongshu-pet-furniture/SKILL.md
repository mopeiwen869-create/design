## 四、Round 1：近半年爆款搜索

### 4.1 目标

抓取用户指定数量的有效案例（默认 100 条）。

### 4.2 时间限制

最近 **180 天**（6 个月）。

**必须使用首次发布时间**，禁止使用编辑时间/修改时间/重新发布时间。

**判断逻辑：**
```
当前日期 = 2026-07-12
最早允许发布时间 = 2026-01-12

if 首次发布时间 < 最早允许发布时间:
    discard（丢弃）
```

### 4.3 搜索执行

每个关键词的执行步骤：

```
1. 打开搜索页：
   https://www.xiaohongshu.com/search_result?keyword=<URL编码关键词>

2. 等待页面加载完成（DOM 稳定）

3. 点击"最新"排序标签

4. 等待排序切换完成

5. 滚动采集：
   for 轮次 in 1..15:
       采集当前可见的帖子卡片
       向下滚动 ≈ 窗口高度 × (0.7 + random × 0.3)
       wait 1.5-3s（随机）
       if 连续 3 轮没出现新帖子 → 停止滚动
   end
```

### 4.4 帖子卡片提取字段

```json
{
  "title": "标题文本",
  "note_id": "帖子ID（从URL提取）",
  "note_url": "完整帖子URL",
  "likes": "点赞数",
  "relative_time": "相对时间标签（如'1个月前'）",
  "author_name": "作者昵称",
  "author_avatar": "头像URL"
}
```

### 4.5 Round 1 排序优先级

通过第一段预筛的帖子，按以下优先级排序进入详情采集：

```
第一优先：收藏数量（高收藏 = 购买参考价值）
第二优先：评论数量（高评论 = 用户兴趣强）
第三优先：点赞数量
第四优先：商业购买意愿信号
```

### 4.6 第一段预筛（卡片级，零开销）

对搜索结果列表做粗筛：

```
时间粗筛：
  相对时间含"刚刚""小时前""天前" → ✅ 保留
  相对时间含"周前"且数字 ≤ 12 → ✅ 保留
  相对时间含"个月前"且数字 ≤ 6 → ✅ 保留
  其他 → ❌ 跳过

点赞粗筛：
  如果列表 ≥ 30 条，取点赞数前 60%
  如果列表 < 30 条，全部保留
```

### 4.7 第二段筛（详情页快速检查）

只有通过第一段筛的帖子才开详情页：

```
for each 候选帖子:
    1. 打开帖子详情页
    2. 等待加载完成
    3. 提取精确首次发布时间
    4. 如果发布时间 > 180天前 → 关闭，跳过
    5. 提取收藏数
    6. 通过 → 进入第三段深度采集
end
```

### 4.8 第三段深度采集

通过第二段筛的帖子，采集完整数据（见第五节）。

### 4.9 Round 1 停止条件

```
累计有效案例 >= 用户指定的 target 数量 → 跳转到 Round 2（如果 mode=A）
或
所有关键词搜索完毕 → 跳转到 Round 2（如果 mode=A 且数量不足）
或
触发熔断 → 直接进入分析阶段
```

## 五、Round 2：扩展搜索（仅 mode=A 且 Round 1 不足时）

### 5.1 触发条件

```
Round 1 有效案例数量 < 用户指定的 target
```

### 5.2 目标

补足累计达到用户指定的 target 数量。

### 5.3 策略变化

Round 2 允许放宽时间限制，但必须满足：

**评论活跃度过滤：**

```
分析帖子的评论发布时间（comment_publish_time）
计算：最近180天评论占比 = 最近180天评论数 / 总评论数

保留条件：最近180天评论占比 >= 30%

说明：即使帖子发布时间较早，如果最近半年仍有大量互动，
     说明该内容持续具有热度和商业价值。
```

### 5.4 Round 2 优先保留的帖子类型

```
1. 发布时间 > 180天，但最近半年评论占比 ≥ 50%
   → 持续热门，用户持续咨询

2. 发布时间 > 180天，但最近半年有购买意向评论
   → 有持续商业价值

3. 高互动但被 Round 1 时间过滤掉的帖子
   → 可能是经典爆款
```

### 5.5 Round 2 停止条件

```
累计有效案例 >= 用户指定的 target → 进入分析阶段
或
候选池耗尽 → 以当前数量进入分析阶段
或
触发熔断 → 以当前数量进入分析阶段
```

## 八、爆款评分模型（HotScore）

### 8.1 计算公式

```
HotScore = 0.35 × 收藏分 + 0.25 × 评论分 + 0.20 × 点赞分 + 0.20 × 购买意愿分
```

### 8.2 各维度归一化

将所有帖子的各指标分别排名，转换为 0-100 分：

```
某帖子的收藏分 = (该帖子收藏数在所有帖子中的排名百分位) × 100
某帖子的评论分 = (该帖子评论数在所有帖子中的排名百分位) × 100
某帖子的点赞分 = (该帖子点赞数在所有帖子中的排名百分位) × 100
```

### 8.3 购买意愿评分

**正向信号词库：**

```
高信号（每命中一个 +15分，上限60分）：
  买了、已下单、求链接、在哪买、多少钱、入手了、已购、
  有链接吗、怎么买、求团购、链接求、同款求、店铺求、
  怎么下单、求购买方式、在哪里入手

中信号（每命中一个 +8分，上限30分）：
  种草了、求推荐、哪里入、什么牌子、这个好、想要、
  好好看、喜欢求、被安利了、想买、安排、准备入手、
  太好看了、收藏了、种草成功

低信号（每命中一个 +3分，上限10分）：
  我家猫需要、准备给猫安排、收藏以后买、有同款吗、
  什么材质、耐用吗、好用吗
```

**负向抵消词（命中任意一条 → 购买意愿分归零）：**
```
别买、避雷、后悔、退了、退货、踩雷、不推荐、智商税、
闲置了、质量问题、坏了、不值得、劝退、别入、翻车、差评
```

**计算逻辑：**

```python
def calc_buying_score(comments):
    if any(neg in comment for comment in comments for neg in NEGATIVE_WORDS):
        return 0
    
    score = 0
    for comment in comments:
        for word in HIGH_SIGNALS:
            if word in comment: score += 15
        for word in MEDIUM_SIGNALS:
            if word in comment: score += 8
        for word in LOW_SIGNALS:
            if word in comment: score += 3
    
    return min(score, 100)  # 上限100
```

### 8.4 最终评分

```
HotScore = 0.35 × 收藏分 + 0.25 × 评论分 + 0.20 × 点赞分 + 0.20 × 购买意愿分
```

按 HotScore 降序排列，得到 TOP 排名。

## 九、爆款内容分析模块

对 TOP 帖子（取 HotScore 前 20 或所有帖子的前 20%，以较大者为准）逐篇分析。

### 9.1 标题分析

**标题结构分类：**

| 类型 | 特征 | 示例 |
|------|------|------|
| 痛点型 | 描述养猫困难/需求 | "猫咪没有自己的房间怎么办" |
| 结果型 | 展示成果+价格 | "3000元打造猫别墅 太值了" |
| 反差型 | 对比前后/新旧 | "普通猫窝 vs 猫别墅 区别太大了" |
| 身份型 | 强调专业身份 | "设计师给猫做了一套房" |
| 故事型 | 讲述过程 | "为了猫改造了整个家" |
| 数字型 | 用数字吸引 | "花了2万给猫做了个别墅" |
| 推荐型 | 推荐/测评 | "猫别墅推荐 这5款闭眼入" |

**分析输出：**
- 各类型占比
- 最有效的标题模式
- 标题平均字数
- 高频词 TOP20

### 9.2 封面分析

分析第一张封面图：

| 维度 | 分析内容 |
|------|---------|
| 主体 | 猫/产品/空间/人物/混合 |
| 构图 | 近景/中景/远景/俯拍/平拍 |
| 色调 | 暖色/冷色/中性/高饱和 |
| 文字 | 有无文字 overlay、字体风格 |
| 场景 | 室内/室外/工作室/展厅 |

**分析输出：**
- 最常见的封面类型
- 点击率最高的封面特征

### 9.3 正文分析

**正文结构拆解：**

**可用 → 生成 .docx：**

使用 python-docx 将 Markdown 报告转换为格式化的 Word 文档。转换规则：

```python
# 转换逻辑：
1. # 标题 → Heading 1
2. ## 标题 → Heading 2
3. ### 标题 → Heading 3
4. 表格 → Word 表格（带边框，表头加粗）
5. **加粗** → 粗体
6. 正文 → Normal 样式
7. 代码块 → 等宽字体灰色背景
8. emoji 保留原样
```

**生成脚本：**

```python
import sys
from docx import Document
from docx.shared import Pt, Inches, RGBColor
from docx.enum.text import WD_ALIGN_PARAGRAPH
from docx.enum.table import WD_TABLE_ALIGNMENT

def md_to_docx(md_path, docx_path):
    doc = Document()
    
    # 设置默认字体
    style = doc.styles['Normal']
    font = style.font
    font.name = '微软雅黑'
    font.size = Pt(11)
    
    with open(md_path, 'r', encoding='utf-8') as f:
        lines = f.readlines()
    
    in_table = False
    table_rows = []
    
    for line in lines:
        line = line.rstrip('\n')
        
        # 表格处理
        if '|' in line and line.strip().startswith('|'):
            cells = [c.strip() for c in line.split('|')[1:-1]]
            # 跳过分隔行
            if all(set(c) <= set('- :') for c in cells):
                continue
            table_rows.append(cells)
            in_table = True
            continue
        elif in_table:
            # 输出表格
            if table_rows:
                table = doc.add_table(rows=len(table_rows), cols=len(table_rows[0]))
                table.style = 'Table Grid'
                for i, row_data in enumerate(table_rows):
                    for j, cell_text in enumerate(row_data):
                        cell = table.cell(i, j)
                        cell.text = cell_text
                        if i == 0:
                            for p in cell.paragraphs:
                                for run in p.runs:
                                    run.bold = True
                doc.add_paragraph('')
            table_rows = []
            in_table = False
        
        # 标题
        if line.startswith('# '):
            doc.add_heading(line[2:], level=1)
        elif line.startswith('## '):
            doc.add_heading(line[3:], level=2)
        elif line.startswith('### '):
            doc.add_heading(line[4:], level=3)
        elif line.startswith('#### '):
            doc.add_heading(line[5:], level=4)
        # 代码块
        elif line.startswith('```'):
            pass  # 简化处理
        # 普通段落
        elif line.strip():
            p = doc.add_paragraph(line)
    
    # 处理末尾表格
    if table_rows:
        table = doc.add_table(rows=len(table_rows), cols=len(table_rows[0]))
        table.style = 'Table Grid'
        for i, row_data in enumerate(table_rows):
            for j, cell_text in enumerate(row_data):
                cell = table.cell(i, j)
                cell.text = cell_text
    
    doc.save(docx_path)
    print(f'Word 文档已生成: {docx_path}')

md_path = sys.argv[1]
docx_path = sys.argv[2]
md_to_docx(md_path, docx_path)
```

## 十二、CDP 操作要点

### 12.1 Tab 管理

```
- 全程在后台 tab 操作，不干扰用户浏览器
- 每次搜索用 /new 打开新 tab
- 搜完后用 /close 关闭 tab
- 任务完成后关闭所有创建的 tab
- 不要一次开多个 tab 并行操作
```

### 12.2 节奏控制

```
- 每次 DOM 操作后 wait 1.5-4s（随机）
- 滚动之间 wait 1.5-3s
- 不同关键词之间 wait 5-8s
- 详情页之间 wait 3-5s
- 随机间隔公式：1500 + Math.random() * 2500（毫秒）
```

### 12.3 反爬策略

```
1. 优先使用 /clickAt（真实鼠标手势）而非 /click（JS 点击）
2. 不固定滚动距离：每次 = 窗口高度 × (0.7 + Math.random() * 0.3)
3. 搜索结果每次采集后随机打乱采集顺序
4. 模拟真人行为：
   - 偶尔在页面停留较久（3-6s）
   - 偶尔往回滚动一点
   - 偶尔点击无关元素再返回
5. 某页面加载超过 15s 无响应 → 关 tab 重试 1 次 → 仍失败则跳过
6. 连续收到空白页或验证码 → 停止所有操作，提示用户
```

### 12.4 页面结构自适应

小红书前端 DOM 可能随时改版。操作时：

```
1. 先用 /eval 打印页面关键结构，找到实际的数据元素
   例如：document.querySelector('body').innerHTML.substring(0, 5000)
2. 根据实际 DOM 调整选择器
3. 如果页面结构与预期不符，不要反复试，改策略
4. 优先使用 data 属性和语义化选择器，而非 class 名
```

## 十三、熔断机制

### 13.1 熔断触发条件

```
满足任意一条即触发熔断，停止采集：

1. 总页面交互数 > 300 次
2. 已开详情页数 > (target × 1.5)
3. 连续 3 个关键词搜不到结果
4. 连续 5 个详情页加载失败
5. 用户手动中断
```

### 13.2 熔断后处理

```
1. 停止所有采集操作
2. 用已采集的数据继续执行分析
3. 在报告中标注"本次采集因熔断提前终止，实际采集 N 条"
4. 关闭所有打开的 tab
```

## 十四、错误处理

| 场景 | 处理方式 |
|------|---------|
| 搜索结果为空 | 尝试 2 次 → 仍空则跳过该词 |
| 详情页 404 | 跳过该帖子 |
| 作者主页无法加载 | 跳过账号识别，标记"未知" |
| 评论区域未加载 | 滚动到底部一次 → 仍无则标记"评论未获取" |
| CDP 断开 | 尝试重新连接 → 失败则提示用户重启浏览器 |
| 验证码出现 | 停止操作，提示用户手动处理 |
| 词表全跑完但不足 target | 以当前数量继续分析，报告中说明 |
| 页面结构变化导致选择器失败 | 先探索 DOM 结构，调整选择器后重试 1 次 |

## 十五、执行流程总览

```
0. 风险告知 → 用户确认
1. 收集参数（keyword, target, mode）
2. 关键词扩展 → 生成搜索词列表
3. Round 1 遍历搜索词
   ├─ 3.1 搜索页 → "最新"排序 → 滚动加载
   ├─ 3.2 卡片级预筛（时间+点赞）
   ├─ 3.3 详情页快速筛（精确时间+收藏）
   ├─ 3.4 深度采集（正文+标签+评论+作者）
   ├─ 3.5 动态关键词扩展（mode=A）
   ├─ 3.6 休息 5-8s → 熔断检查 → 下一个词
   └─ 3.7 达到 target → 跳到步骤 4
4. Round 2 补充（如果 mode=A 且不足 target）
   ├─ 4.1 扩大时间范围
   ├─ 4.2 评论活跃度过滤
   └─ 4.3 达到 target 或候选池耗尽
5. 去重合并
6. 账号分类
7. HotScore 评分
8. 内容分析（TOP 帖子）
9. 反向内容生成
10. 输出报告（MD + JSON + CSV）
11. 关闭 tab，输出文件路径
```
